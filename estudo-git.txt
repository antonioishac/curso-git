- Clonando um repositório
	git clone https://github.com/antonioishac/EstudyCollections.git
	
- verificando a versão da tag
	git tag
	
- trocando de versão
	git checkout v0.1
	
- diferença entre versões
	git diff v0.1 v0.2
	
- comando para dizer que uma pasta será um repositório do git
	git init
	
- verificando os arquivos que pertence ao git no repositório
	git lf-files
	
- verificando o status dos arquivos no git
	git status

- informando ao git que devemos adicionar um arquivo ao repositório
	git add index.html
	
- informando ao git todos os arquivos
	git add .

- configurando variaveis
	git config user.name "Antonio Ishac"
    git config user.email "antonioishac@gmail.com"

- configurando variaveis globalmente
	
	git config --global user.name "Antonio Ishac"
	git config --global user.email "antonioishac@gmail.com"
	
- realizando commit
	git commit -m "primeiro commit"
	para verificar, realize o comando "git status" para verificar o estado.

- alterando o arquivo index.html, "git status", verifica que realmente modificamos esse arquivo.
  Precisamos rodar o comando "git add index.html" novamente.
  Feito isso, estamos pronto para realizar o commit " --> git commit -m "conteudo da pagina index" <--".
  
  git commit -a

 
- A opção -a inclui imediatamente no commit todos os arquivos modificados ou removidos! 
  Porém, ela não adiciona os arquivos novos. Podemos combinar a opção de mensagem com:
  git commit -a -m 'Mensagem de commit' 
  para não precisar escrever a mensagem no editor padrão do console.
  
- mandando nossos arquivos para o github
	- criando o nome do repositório no github
	- Agora que temos um repositório remoto, 
	  precisamos configurá-lo para sincronizar com o repositório local. 
	  Para isso, precisamos ir ao diretório do repositório local no terminal e executar o seguinte comando:
	  " git remote add origin https://github.com/antonioishac/curso-git.git "
	- enviando os commits relalizados para o repositório:
	  git push origin master
	  
- Mostrando o log dos commits
	git log
	
- Mostrando opções detalhadas dos commits
	git whatchanged
	
- Mostrando opções detalhadas dos commits, é possível visualizar quais as linhas que foram modificadas em cada commit do nosso projeto.
	git whatchanged

- comando para criar uma nova branch
	git branch design

- comando para listar as branches locais
	git branch

- comando para trocar de branch
	git checkout design
	
- Uma tarefa bem comum é a criação de uma nova branch seguida da troca para essa branch criada. 
  Poderíamos fazer isso com a sequência de comandos git branch nomeDaBranch e git checkout nomeDaBranch. 
  Para facilitar, o Git nos fornece uma opção do comando git checkout que realiza este trabalho. Qual a opção que deve ser passada para realizar tal tarefa?
	git branch -b design
	
- comando para remover uma branch
	git branch -d design
	
	caso não esteja sincronozada, e haja a necessidade de exclusão, podemos forçar
	git branch -D design
	
- criando uma branch remota
	git push -u origin design
	
- comando para listar somente as branch remotas
	git branch -r
	
- comando para listar as branch locais e remotas
	git branch -r
	
- Uma sequência de tarefas bem comum referente à cópia de uma branch localizada num repositório remoto é:
	Criação de uma branch local com o mesmo nome da branch remota
	Mudança para essa nova branch criada
	Criação de link entre a branch local e remota
  Para evitar toda essa tarefa, o git nos fornece um atalho para evitar esse trabalho todo com uma opção do comando git checkout. Que opção é essa?
	git checkout -t origin/design
  Com o comando git checkout -t origin/design, todo esse trabalho é feito automaticamente para a branch chamada design.

- comando para remover uma branch remota
	git push -d origin design
	
- comando para saber se foram criadas branches novas no repositório remoto
	git fetch origin
  Este comando verifica todas as atualizações que foram realizadas no repositório de atalho origin.
